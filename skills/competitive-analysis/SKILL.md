# 竞品分析 Skill

分析分 3 个阶段，按顺序执行：

## Phase Map

```
用户说"分析XX的YY功能"
  │
  ├─→ 阶段 1: 分析发起 → 确认竞品+功能+体验方式（4选1）
  ├─→ 阶段 2: 截图采集 → 5条纪律，步步截图步步验证
  └─→ 阶段 3: 报告生成 → 总-分结构 + 4道质量门禁
```

---

## 阶段 1：分析发起

### Step 1 — 确认目标
复述确认："你要分析的是 **{竞品名}** 的 **{功能名}**，对吗？"

### Step 2 — 确认体验方式（4选1，可多选）
```
请确认竞品体验方式：
1. 📤 你提供截图 — 你截图发我分析
2. 🔍 我搜索网上资料 — 官方文档/帮助中心/评测
3. 🌐 我Web端体验并截图 — 浏览器操作竞品网站
4. 📱 手机连接自动截图 — USB连接，ADB操控

请选择（可多选，如 2+3）：
```

### Step 3 — 选了AI截图→先搜索再操作
搜索 `"{竞品名} {功能名} 入口路径"` 和 `"{竞品名} {功能名} 操作步骤"`。
搞清楚入口、步骤、前置条件后再截图。
报告必须用亲自截的实际截图，不得用他人截图。

---

## 阶段 2：截图采集（7条纪律）

**A. 步步截图不跳步。** 路径N个节点 = N张图，缺一不可。

**B. 截图后立即验证内容。** 每截一张检查页面是否匹配。截到锁屏/桌面/无关页面→绝对不允许标注为流程步骤。截图内容必须与标注的流程步骤严格一致。

**C. 连续失败必须求助（最重要）。** 连续2次截到相同页面→立即停止，报告"卡在XX页，请确认"；连续2次截到不相关页面→立即停止，报告"预期XX但截到YY"。**绝对禁止用锁屏截图冒充流程截图。**

**D. 操作后等页面稳定。** 点击→等1.5秒→确认变化→再截图。

**E. 禁止提前说完成。**

**F. 文件命名反映路径。** `01_首页.png` / `02_搜索页.png`

**G. 截图去重检测。** 每截一张与上一张对比。内容相同→操作没生效→回到C求助。

### ADB 截图
1. `uiautomator dump` → `cat` 读 UI 树 → 找元素 bounds → 计算中心点 → `input tap`
2. `screencap` 截图 → `pull` 拉取 → 验证内容
3. 将截图保存到 `screenshots/<竞品>/<功能>/`

### Web 截图（Playwright）
1. `page.goto(url)` → `waitUntil: 'networkidle'`
2. `page.screenshot()` → 验证内容
3. 点击操作 → 等待 → 再截图

## 阶段 3：报告生成

### 总-分结构

**分（逐个竞品）：**
1. 路径还原：`→` 箭头链，每段标截图编号，不确定标`[待确认]`
2. 步骤表：`| 步骤 | 截图 | 🔍观察 | 操作 | 阶段 |`
3. 分支场景：触发条件+路径+截图覆盖，未覆盖标`[待确认]`
4. Mermaid流程图：实线=🔍确认，虚线=💡推测(<10%)，无截图不画
5. 亮点痛点：各≤3条，标截图来源

**总（跨竞品对比）：**
1. 对比总表：行=维度，列=竞品，每格标来源，无来源填`[待确认]`
2. 分类总结：只总结已确认共性
3. 差异化分析：标差异来源
4. 设计建议：逐条标对标来源

### 零幻觉原则
1. 每结论必须有截图依据 2. 不确定标`[待确认]` 3. 区分🔍观察vs💡推测(<10%)
4. 禁止虚构 5. 有疑问就提问 6. 模糊就写"不清晰无法确认"

### 5道质量门禁（每条必须输出可见证据，不是心里想、是写出来）

**门禁1—流程图-截图映射表**：Mermaid每个节点必须标截图编号，无截图节点从流程图删除。强制输出映射表逐行检查。
**门禁2—对比数据来源表**：参与对比的竞品必须声明数据来源，无截图的竞品不得参与对比。
**门禁3—设计建议对标表**：每条建议标注对标竞品+截图，无对标删除。
**门禁4—截图覆盖声明**：以流程图节点数为分母，覆盖<60%不得输出报告正文。
**门禁5—截图数量下限**：每个竞品至少5张截图才允许生成分析。

### 截图覆盖声明（强制）
```
| 竞品 | 已截步骤 | 缺失步骤 |
已覆盖: X/N (X%)
```
覆盖<60%主动建议补充。
