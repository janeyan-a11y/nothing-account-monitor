# 竞品分析维度

每次竞品分析需覆盖以下维度：

1. **入口路径**：从主页/设置页出发，到达该功能的最短路径
2. **开启流程**：如何开启该能力（逐步说明）
3. **关闭流程**：是否支持关闭、关闭路径、关闭后的状态
4. **使用流程**：开启后如何触发使用、使用中的交互
5. **分支场景**：不同前置条件下的分支路径（如 已登录/未登录、已绑定/未绑定、首次使用/再次使用 等）
6. **异常处理**：网络错误、账号错误、服务器错误等异常状态的处理
7. **体验亮点**：该竞品做得特别好的地方
8. **体验痛点**：该竞品体验差或缺失的地方

# 输出格式规范

- **路径表示**：用 `→` 箭头链式表示，如 `设置 → 账号 → Google账号 → 关联账号`
- **流程图**：统一用 Mermaid `flowchart TD` 语法，必须包含所有分支条件
- **对比表**：统一用 Markdown 表格，行=维度，列=竞品
- **语言**：所有分析用中文输出
