账号定位与 2026 核心目标

一、账号是什么？—— 生态核心用户资产与变现枢纽

打通生态： 依托统一身份，无缝打通软硬件服务，筑牢生态护城河。

商业赋能： 锁定用户留存，沉淀核心数据资产，带动硬件复购与订阅增收，深度挖掘用户的长期商业价值。

二、账号2026年目标与挑战？

目标：支撑订阅能力上线及 MCP / Napkin 等业务全面接入。

挑战：陈旧的 PHP 架构存在严重安全隐患，难以支撑金融级的安全要求。

上半年总结与下半年规划

H1（上半年）

还历史欠账，完成底层重构，实现从“能用”到“安全、可扩展”的跨越。

☕ 1. 后端重构（最核心底座）

全面废弃陈旧 PHP 架构，采用现代化 Java 治理，实现接口标准化与资源灵活扩缩容，彻底清除了以往接口非标、多业务共用 Token 等安全隐患，为后续接入 MFA（多因素认证）等现代安全标准扫清障碍。

📱 2. App 端架构革新

摒弃多端分离，采用一套代码（Flutter）跨平台复用，大幅降低维护成本，并全面接入了 2FA（双因素认证）与 SSO 能力。

🌐 3. Web 端（海外）升级

从 0 到 1构建了标准化的海外账号中心；采用分步式极简表单设计，有效降低流失率。

🛡️ 4. 全链路安全破局

安全能力从无到有，上线了设备可信验证、非授信设备二次验证强制触发机制以及 2FA 全流程覆盖，目前安全模型底座已构建 65%。

🛡️ 风控体系加固

部署业务入口二次验证能力，强化第三方账号安全体系，建立中国区/海外的频率限制与反作弊规则，严防撞库与盗刷。

🔑 安全验证创新

新增 Passkey（无密码）验证支持。利用本地生物识别进行端云并行预校验，在保证极高安全性的同时，实现极简无感认证，降低支付环节的身份验证流失率。

数据互联与精细化运营

接入内部数据系统（行为/品质/售后），建立账号数据看板平台，实现账号属性与业务场景（支付/订阅行为）的深度同频。

隐私安全合规

全面满足监管要求并规避隐私风险，主要通过完善全链路数据合规采集与关键日志记录、优化支持单业务独立注销的精细化销户体验、完成账号能力的Google Play合规上架等实现。
